## 核心主題

本影片由 JG 以台指期空單操作為例，系統性地拆解一筆完整交易的三個生命週期——**建倉期、加碼期、停利期**——並為每個階段設定明確的心理定位與操作原則，幫助交易者在盤勢混亂時，仍能依循結構化框架冷靜執行，避免因情緒慌亂而做出破壞整體操作邏輯的決策。

---

## 關鍵觀點

- **「建倉期就是來賠錢的。」**
  進場時本就不知道行情是否如預期走，因此建倉期的核心任務是**設好停損、等待驗證**，不是來賺錢的。

- **「加碼期的重點是拿賺到的錢去拼未來的獲利。」**
  加碼不是用本金硬加，而是用浮動獲利去擴大部位，性質是「拿賺的去賭未來」。

- **「加碼期：有賺不能賠。」**
  一旦進入加碼期，平均成本絕對不能變成虧損，這是加碼期的鐵律。

- **「停利期你就隨便了。」**
  到了停利期，操作自由度大幅提升，心態應從「有賺不能賠」升級為「多賺一點再出場」，讓獲利更充分實現。

- **「操作其實是基於一個你舒服的模型下去動作，怕的是你不舒服。」**
  交易框架設計的核心目的，是讓操作者在任何階段都能維持心理舒適度，降低情緒干擾決策的機率。

- **「停損沒有到，你是不是都不要動作？」**
  只要停損點未被觸及，就應遵守紀律按兵不動，不因短暫反彈或被軋空而恐慌操作。

- **「不要讓賺錢的變成賠錢的。」**（重複強調）
  這是貫穿加碼期最核心的風控底線。

## 實操規則

### 前提設定（以台指期空單為例）
| 項目 | 數值 |
|------|------|
| 建倉位置 | 約 15,200 點 |
| 停損幅度 | 400 點 |
| 最低停利目標 | 1,200 點 |
| 風報比要求 | **最低 1:3**（損400：利1200） |

---

### 階段一：建倉期
- **資金分配**：以總資金的一部分（例：300萬中的100萬）進行建倉，其餘資金保留。
- **進場方式**：可逆向分批（例：分3次或5次建倉），取得合理平均成本。
- **操作原則**：
  - 進場後**設好停損，價格未觸停損前不動作**。
  - 心理定位為「來繳稅的」、「來點人事成本的」——接受可能虧損的事實。
  - 建倉部位應保守，不過度重押。

---

### 階段二：加碼期
- **進入時機**：行情順方向走，**獲利約達 300～500 點**時開始考慮加碼（不是等到停利目標才加）。
- **資金來源**：動用建倉資金以外的部位，且應以**帳面浮盈**作為加碼的底氣。
- **操作原則**：
  - 加碼後，**整體部位的平均成本不可變成虧損**。
  - 不限制加碼量，但必須確保「有賺不能賠」。
  - 即使後續盤勢回調、加碼期以小獲利出場，仍屬**好操作**，不需懊悔。

---

### 階段三：停利期
- **進入時機**：獲利達 **1,000～1,400 點以上**（視個人設定）。
- **操作原則**：
  - 心態升級為「**多賺一點再出場**」，不再拘泥於「有賺不能賠」的底線。
  - 此階段自由度高，可依個人判斷靈活出場。
  - **警告**：停利期不應因新情報或貪念而重新加重倉位，否則心態會又退回加碼期的緊繃狀態，易造成心理不適。

---

## 重要比喻或案例

- **「殺進去就是來繳稅的」／「來點人事成本的」**
  → 用來描述建倉期的心理定位：虧損是做交易必要的成本支出，不是失敗，接受它才能冷靜執行。

- **「300萬拿100萬建倉」**
  → 具體說明資金分層使用的概念：不是全倉進去，而是留有餘裕，讓後續加碼有子彈。

- **「床底下的私房錢 all in 加碼都沒關係」**
  → 強調加碼期的「膽量」來自已有浮盈墊底，但前提是平均成本不能翻虧。

- **「好不容易到停利期，你又更貪心，可能看了一個情報……你又跟他 all in」**
  → 警示停利期最常見的陷阱：資訊驅動的衝動加倉，打亂整個操作結構。

- **「本來賺很多，變成只賺一點點」（JG 自身經驗）**
  → JG 自承曾在停利期過度大膽，結果獲利大幅縮水，以親身案例說明分期管理的必要性。

## 適用情境

| 情境 | 說明 |
|------|------|
| **盤勢劇烈波動、方向混亂** | 最直接適用，幫助交易者在慌亂中確認自己處於哪個「時期」，依規則行動而非情緒反應。 |
| **持有空單遭遇反彈（被軋）** | 用「停損未到 → 不動作」的原則穩住心態，避免恐慌平倉。 |
| **趨勢行情順風時** | 加碼期邏輯最能發揮效益，拿浮盈擴大部位、放大順勢獲利。 |
| **大行情末段** | 停利期框架協助鎖利，避免因貪婪吐回大部分獲利。 |
| **新手至老手皆適用** | JG 明確指出此框架「股票、期貨、黃金一體適用」，不限商品與資歷。 |
